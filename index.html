About

This was a high school project of mine, where we were required to make some sort of voting system... and Blue Ballot was my solution. Blue Ballot or BB for short (as it will now be called in the rest of this document), is a suite of 4 applications, written completely in Java, designed to manage and facilitate a voting process. Whether this might be for a school council, company board, local municipality or anything really. When I designed it I had local municipal elections in mind (however I doubt very much that this is an appropriate use, due to lack of testing) and it is for this reason that the system uses 2 sets of passwords (Governmental and United Nations).

Platform Specific Releases

If you want platform specific releases then you will have to compile them yourself. This is done using the ant build script provided. There are individual ones for each app as well as a global one for the suite.

Note: To compile you will need to have the BBLibrary.jar file in /dist/lib if its not already there. As well as the BBLibrary.jar file you will need the libs required to build the platform specific executables. Look in build.xml for more information about where to get these.

When making platform specific releases and you want the Java Runtime Environment to be bundled with the app you must specify where that JRE is. To do this edit build.xml to point to the correct place. For example:

<property name="myApp.jre" value="/Library/Java/JavaVirtualMachines/jdk1.7.0_12.jdk/Contents/Home"/>

Alternatively if you do not wish to bundle the JRE then comment out any lines containing <runtime dir="${myApp.jre}" />

If you have netbeans then it will be really easy for you as each app as well as the library is a netbeans project. Just open them in netbeans, open the file tab, then right click the build.xml file choose run targets -> other targets -> the platform you want (eg. mac). This will compile to distributables. Note: At the moment the netbeans project has the BBLibrary netbeans project as a dependancy. Feel free to remove this if you have the BBLibrary.jar file in /dist/lib

To build without netbeans (requires ant) cd /Project/Root/ ant <yourplatform> where is mac or windows

Project Breakdown

Blue Ballot Registration App

Provides voters with a unique key that they can use to cast their virtual ballot.

Blue Ballot Voting App

Allows voters to vote over the air. It also provides news, updates, candidate information and various other features to the voter.

Blue Ballot Server App

Used to control the server service/daemon. The server accepts connections, ballots, requests, etc. as well as serves information to the other apps such as news, updates, candidate information etc. This is the central hub that all the other apps connect to.

Blue Ballot Admin App

Allows the election admin to setup and manage the election process. It connects to the various servers and allows the admin to control each election separately.

Blue Ballot Library

The lib for the other apps. Needed for compile purposes ONLY. Note that this is not a standalone part of the suite.
